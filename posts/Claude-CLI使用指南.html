On this page

把 Claude 放进终端，让它能读项目、改文件、跑命令、做 review

前言 ​

Claude CLI 通常指 Claude Code CLI。它不是普通网页聊天框，而是运行在终端里的 AI 编程助手。

你可以把它理解成：

  • claude：打开一个交互式 AI 编程会话
  • claude -p "问题"：问完就退出，适合脚本/管道
  • /xxx：在会话内部控制 Claude Code，比如清空上下文、切换模型、查看 diff
  • 权限弹窗：Claude 想改文件、跑命令、访问网络时，通常会先问你

本文重点放在两件事：

  1. 指令怎么用：CLI 命令、flags、会话内 / 指令
  2. 运行时怎么看懂：颜色、英文状态、权限提示、工具调用含义

说明：Claude Code 更新比较快，具体可用命令会受版本、系统、账号套餐、企业策略影响。遇到差异时优先看 claude --help、/help 和官方文档。

第一部分：基础上手 ​

启动与常用方式 ​

交互式启动 ​

bash
claude

含义：进入 Claude Code 的交互式会话。适合长期改项目、调 bug、写文档。

进入项目目录后再启动更自然：

bash
cd /path/to/your/project
claude

Windows PowerShell 示例：

powershell
cd D:\develop\your-project
claude

带第一句话启动 ​

bash
claude "explain this project"

含义：启动交互式会话，并把 "explain this project" 当作第一条消息发给 Claude。

常用例子：

bash
claude "帮我分析这个项目的技术栈和入口文件"
claude "检查当前项目有没有明显的启动问题"

问完就退出：-p ​

bash
claude -p "explain this function"

-p 是 --print 的缩写。它不会进入交互界面，而是输出结果后退出。

适合：

  • 写脚本
  • 处理日志
  • CI 中做一次性分析
  • 和管道配合使用

管道输入 ​

macOS / Linux / Git Bash：

bash
cat logs.txt | claude -p "总结这里面的错误原因"

PowerShell：

powershell
Get-Content .\logs.txt | claude -p "总结这里面的错误原因"

把命令输出交给 Claude 分析：

bash
git diff | claude -p "帮我 review 这次改动，重点看潜在 bug"

PowerShell：

powershell
git diff | claude -p "帮我 review 这次改动，重点看潜在 bug"

继续最近会话：-c ​

bash
claude -c

含义：继续当前目录下最近的一次 Claude 会话。

也可以在非交互模式下继续：

bash
claude -c -p "继续检查刚才的类型错误"

恢复指定会话：-r ​

bash
claude -r "auth-refactor"

或者带一条新消息恢复：

bash
claude -r "auth-refactor" "继续完成这个 PR"

-r 是 --resume 的缩写，可以按 session id 或 session name 恢复。

更新与版本 ​

查看版本：

bash
claude -v

更新：

bash
claude update

安装或重装稳定版：

bash
claude install stable

常用 flags 速查 ​

flag 是启动 claude 时加在后面的参数。

--model：指定模型 ​

bash
claude --model sonnet
claude --model opus

也可以使用完整模型名，但完整名字会随官方版本变化。日常更推荐用 sonnet、opus 这类别名。

--permission-mode：指定权限模式 ​

bash
claude --permission-mode plan
claude --permission-mode acceptEdits

常见值：

模式含义
default默认模式，读文件通常不问，改文件/跑命令会问
acceptEdits自动接受文件编辑，适合你愿意事后看 diff 的场景
plan计划模式，只分析和写计划，不直接改代码
auto自动执行更多动作，但有额外安全检查
dontAsk不弹权限询问，只执行预先允许的动作
bypassPermissions跳过权限层，危险，只适合隔离容器/虚拟机

--add-dir：额外授权目录 ​

bash
claude --add-dir ../shared-lib

默认 Claude 主要操作当前项目目录。需要它读取/编辑旁边的目录时，用这个参数。

--mcp-config：加载 MCP 配置 ​

bash
claude --mcp-config ./mcp.json

只使用这份 MCP 配置，忽略其他来源：

bash
claude --strict-mcp-config --mcp-config ./mcp.json

--output-format：指定输出格式 ​

主要配合 -p 使用：

bash
claude -p "总结这个项目" --output-format text
claude -p "总结这个项目" --output-format json
claude -p "总结这个项目" --output-format stream-json

含义：

格式适合场景
text人看，最普通
json脚本解析一次性结果
stream-json流式读取，适合更复杂的自动化

--max-turns：限制轮次 ​

bash
claude -p --max-turns 3 "检查这个问题"

适合自动化脚本，防止 Claude 一直循环执行。

--tools：限制可用工具 ​

bash
claude --tools "Read,Grep,Glob"

含义：本次会话只允许 Claude 使用指定工具。比如只让它读和搜索，不让它编辑或跑命令。

常见工具名：

工具含义
Read读文件
Edit精确编辑文件
Write新建或覆盖文件
Bash执行 shell 命令
PowerShell执行 PowerShell 命令
Grep搜索文件内容
Glob按文件名模式找文件
WebFetch读取网页
WebSearch搜索网页

--name：给会话命名 ​

bash
claude --name "blog-article"

之后可以用这个名字恢复：

bash
claude -r "blog-article"

会话里也可以用：

text
/rename blog-article

--verbose：显示更详细输出 ​

bash
claude --verbose

适合排查问题，能看到更完整的 turn-by-turn 输出。

--append-system-prompt：追加系统提示 ​

bash
claude --append-system-prompt "回答时使用中文，代码注释保持简洁"

这个是“追加”到默认 Claude Code 行为后面。一般比 --system-prompt 安全，因为不会替换掉默认的工具和安全规则。

--worktree：隔离工作区 ​

bash
claude --worktree feature-auth

含义：在隔离的 git worktree 中启动 Claude。适合大改动，避免直接把当前工作区搅乱。

第二部分：会话控制 ​

会话内 / 指令速查 ​

进入 claude 交互界面后，以 / 开头的是 slash command。

输入：

text
/

可以打开命令菜单；输入 / 后继续打字可以筛选，例如：

text
/con

常见会筛出 /context、/config 等。

注意：/ 指令必须在一条消息开头。比如 /help 可以识别，请你 /help 不会被当成命令。

日常会话 ​

指令含义
/help查看帮助和可用命令
/exit 或 /quit退出 Claude Code
/clear [name]开新会话，清空当前上下文；旧会话仍可恢复
/resume [session]恢复会话，也叫 /continue
/rename [name]给当前会话命名
/copy [N]复制最近一次或第 N 次 Claude 回复
/export [filename]导出当前对话
/usage查看用量、成本、限制；/cost、/stats 是别名
/status查看版本、模型、账号、连接状态

项目初始化与记忆 ​

指令含义
/init为项目生成 CLAUDE.md 指南
/memory编辑或查看 Claude 的记忆文件
/add-dir <path>当前会话中增加一个可访问目录
/config 或 /settings打开设置界面
/keybindings打开或创建快捷键配置

CLAUDE.md 可以理解成给 Claude 的项目说明书，例如：

  • 项目怎么启动
  • 测试命令是什么
  • 代码风格要求
  • 哪些目录不要碰

模型与思考强度 ​

指令含义
/model [model]切换模型
`/effort [levelauto]`
`/fast [onoff]`

/effort 常见值：

text

high

auto

简单问题用低一点，复杂重构、review、安全分析用高一点。

指令含义
/context [all]用彩色网格查看上下文占用

/btw <question>问一个临时小问题，不写入主对话历史

上下文快满时，优先用：

text
/context

如果确实太长：

text

编码与变更 ​

指令含义
/plan [description]进入计划模式，先分析方案再决定是否改
/diff打开交互式 diff 查看器
/review [PR]本地 review 一个 PR 或当前改动
/security-review分析当前分支未提交改动中的安全风险
/simplify [focus]检查最近改动的复用、质量、效率问题并尝试修复
/rewind回到前面的对话/代码检查点，也叫 /checkpoint、/undo

text
/diff
/review

权限模式详解 ​

Claude Code 不是每个动作都会自动执行。它会根据权限模式决定是否问你。

你可以在会话中按：

text
Shift + Tab

循环切换常见权限模式。也可以启动时指定：

bash
claude --permission-mode plan

default ​

默认模式。

特点：

  • 读文件通常不用问
  • 改文件要问
  • 跑 shell 命令要问
  • 访问网络通常要问

适合：

  • 第一次用 Claude Code
  • 处理敏感项目
  • 不希望它随便动文件

acceptEdits ​

也可以理解成“自动接受编辑”。

状态栏可能会看到：

text
accept edits on

含义：

  • Claude 可以自动创建/修改工作区内文件
  • 一些常见文件系统命令也可能自动批准
  • 其他更危险命令仍可能询问

适合：

  • 你信任当前方向
  • 愿意改完后统一看 /diff
  • 小范围代码改动

不适合：

  • 改生产配置
  • 执行删除、迁移、部署等高风险动作

plan ​

计划模式。

含义：

  • Claude 可以读项目、分析问题、提出方案
  • 不会直接改代码
  • 适合大改动前先想清楚

进入方式：

text
/plan 重构订单模块，把支付逻辑拆出去

或：

bash
claude --permission-mode plan

auto ​

自动模式。

含义：

  • 尽量减少权限弹窗
  • 有额外安全分类器检查动作
  • 适合长任务和低风险任务

注意：

  • 不是所有账号、模型、环境都支持
  • 它减少打扰，但不等于绝对安全
  • 敏感操作仍要自己 review

dontAsk ​

含义：

  • 不弹出询问
  • 只执行已经预先允许的工具
  • 其他需要询问的动作会被自动拒绝

适合：

  • CI
  • 自动化脚本
  • 锁死权限的环境

bypassPermissions ​

含义：

  • 跳过权限提示和安全检查
  • 工具调用会直接执行

这个模式很危险，只建议在：

  • 容器
  • 虚拟机
  • dev container
  • 没有重要文件、没有敏感凭据的隔离环境

启动示例：

bash
claude --permission-mode bypassPermissions

或者：

bash
claude --dangerously-skip-permissions

看到 dangerously 这个词就应该警觉：它的意思就是“危险地跳过权限”。

工具调用相关英文 ​

Claude Code 会显示它用了什么工具。常见工具名如下：

工具名含义你看到时代表什么
Read读文件Claude 正在看某个文件内容
Edit修改文件Claude 准备做局部替换
Write写文件Claude 准备新建或覆盖文件
Bash执行 shell 命令Claude 要跑命令，例如测试、构建、git
PowerShell执行 PowerShell 命令Windows 下跑 PowerShell
Grep搜索内容类似 rg，按文本查
Glob找文件名按 **/*.ts 这类模式找文件
WebFetch读取网页Claude 要访问某个 URL
WebSearch搜索网页Claude 要联网搜索
Agent启动子 agentClaude 要把任务分给另一个 agent
Skill使用技能Claude 加载某个流程/知识技能

看到工具调用时，不要只看名字，还要展开看参数。例如：

  • Bash(npm test)：要跑测试
  • Bash(rm -rf ...)：高风险，谨慎
  • Edit(src/app.ts)：要改某个文件
  • WebFetch(domain:example.com)：要访问某个网站

权限弹窗里的英文 ​

不同版本 UI 文案会变，但意思大致如下：

英文中文解释
Allow / Yes允许这次操作
Deny / No拒绝这次操作
Always allow以后同类操作都允许
Allow for this session当前会话内允许
Accept edits接受文件编辑
Accept all本轮/本会话更自动地接受改动
Review changes查看改动
Permission required需要权限确认
Do you want to proceed?是否继续执行

建议：

  • 读文件、查找、运行测试：通常可以允许
  • 删除文件、批量移动、安装依赖、访问网络：先看清命令
  • 部署、推送、改权限、删库迁移：除非你明确要做，否则拒绝

运行时颜色含义解释 ​

颜色会受终端、主题、/theme、/color、自定义 statusline 影响，所以不要把颜色当成绝对规则。

但日常可以按下面理解。

常见颜色直觉 ​

颜色常见含义
绿色 green成功、通过、已批准、PR approved
黄色 yellow等待、警告、待 review、需要注意
红色 red错误、失败、拒绝、危险、changes requested
灰色 gray草稿、禁用、次要信息、未激活
紫色 purple已合并、特殊状态、强调
蓝色/青色 blue/cyan信息、链接、选中项、提示
橙色/粉色 orange/pink多数只是主题或会话颜色

PR 状态颜色 ​

如果当前分支有关联 PR，Claude Code 可能在底部显示 PR 状态。

官方颜色含义：

颜色PR 状态
Greenapproved，已批准
Yellowpending review，等待 review
Redchanges requested，需要修改
Graydraft，草稿
Purplemerged，已合并

/context 的彩色网格 ​

text
/context

它会显示当前上下文占用。你可以这样理解：

  • 越靠近安全颜色：上下文还比较宽裕
  • 越靠近警告颜色：上下文变重了
  • 如果提示 capacity、warning、heavy usage：该考虑 /compact

常见处理：

text
/compact 保留当前目标、已做决定、已改文件、待办事项和验证结果

status line 颜色 ​

/statusline 可以自定义底部状态行，甚至可以让脚本输出 ANSI 颜色。

比如绿色 ANSI 代码常见写法：

bash
printf '\033[32mOK\033[0m\n'

含义：

  • \033[32m：开始绿色
  • \033[0m：重置颜色

如果你的终端显示乱码，或者颜色影响阅读，可以先切换主题：

text
/theme

或者简化/删除自定义 statusline：

text
/statusline clear

常见问题排查 ​

这里不展开安装和登录问题，只记录已经能进入 Claude Code 后，日常使用时最容易遇到的几个情况。

权限弹窗太频繁 ​

可以按风险从低到高处理：

  1. 对当前动作选择允许
  2. 用 /permissions 给常用安全命令加 allow
  3. 按 Shift+Tab 切到 acceptEdits
  4. 明确知道风险时再考虑 auto
  5. 只有隔离环境才考虑 bypassPermissions

示例：如果你经常允许读文件和跑测试，可以让 Claude 帮你配置：

text
帮我配置 permissions：允许读取项目文件、运行 npm test，但不要允许 push、deploy、rm -rf 这类危险命令。

小结 ​

Claude CLI 的核心不是记住所有命令，而是掌握这条线：

text
启动会话 -> 让它先读项目 -> 用 /plan 控制大改动 -> 看权限弹窗 -> 改完看 /diff -> 必要时 /compact 或 /clear

日常最常用的几个命令：

text
/help
/plan
/diff
/context
/compact
/permissions
/mcp
/usage
/status

把这几个用熟，Claude Code 就不只是“会聊天”，而是能比较稳定地帮你读项目、改代码、跑验证、做总结。
